Showcase 5: Retrieving SQL Data

This example finally demonstrates retrieving data from a SQL server. This data is a backup of the popular AdventureWorks database, which is loaded from a backup and stored on persistent storage in the cloud. So, if something were to go wrong to cause the pods to go down the data is not lost.

Keep in mind that the frontend is still making a call to the backend API, and only the API communicates with the SQL database to retrieve the data. There is a Kubernetes network policy (set-up with Calico) that only allows ingress traffic from API pods to the SQL pod.
